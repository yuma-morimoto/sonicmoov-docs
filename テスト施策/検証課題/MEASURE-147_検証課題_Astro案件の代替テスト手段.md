# MEASURE-147 【検証】Astro案件の代替テスト手段

- Backlog: [MEASURE-147](https://sonicmoov.backlog.jp/view/MEASURE-147)
- 状態: 未対応
- 担当者: 西川真理奈
- 期間: 2026-06-10〜2026-07-31

## 課題

- Astro（website-starter-kit）のテスト環境が不十分。
- Astroコンポーネントテストは技術的な要因で難しく、アコーディオン、スクロール制御などのインタラクションは手動確認に寄っている。

## 検証したい仮説

- Astroコンポーネントテストが難しい場合でも、代替案を調査し、実装可能な確認手段を用意できる。
- uiMatchやVitestによるVRTが、Astro案件のUIテスト・ビジュアル差分検知に有効である。

## 対象案件

- オッズパーク（候補）

## アクション

### コンポーネントテスト代替案の検証

- [ ] Astro案件から、手動確認に頼っているUIまたはインタラクションを1〜2個選ぶ。
- [ ] Astroコンポーネントテストの代替案を調査する。
- [ ] 調査した代替案のうち、実装できそうなものを1つ以上試す。
- [ ] 代替案で確認できる不具合と、確認できない不具合を整理する。

### VRTの有効性検証

- [ ] VRTで確認したいUI差分の対象を決める。
- [ ] uiMatchでFigmaとの差分確認ができるか試す。
- [ ] VitestによるVRTでUI差分を検出できるか試す。
- [ ] uiMatchとVitest VRTで、検出できる差分・検出しづらい差分を比較する。
- [ ] VRTで自動化する範囲と、人間の目視確認に残す範囲を整理する。

## 仮説の有効さを判断する

- [ ] Astroコンポーネントテストが難しい場合の代替確認手段を提示できる。
- [ ] 代替案を実装できる、または実装できない理由を説明できる。
- [ ] uiMatchやVitestによるVRTで、UIの差分や確認漏れに気づける。
- [ ] VRTで検出できる差分と、人間が見るべき差分を分けられる。
- [ ] Astro案件向けの現実的なUIテスト・確認パターンを提示できる。
