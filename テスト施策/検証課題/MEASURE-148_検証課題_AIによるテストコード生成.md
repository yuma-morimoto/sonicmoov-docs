# MEASURE-148 【検証】AIによるテストコード生成

- Backlog: [MEASURE-148](https://sonicmoov.backlog.jp/view/MEASURE-148)
- 状態: 未対応
- 担当者: 森本悠真
- 期間: 2026-06-10〜2026-07-31

## 背景と課題

- 施策へのフィードバックで、「AIを前提としたテスト」としてガイドラインを作ることが指摘された。
- AIを活用したテストコード生成の有効性や、実務レベルの品質を担保できるかが不明瞭なため、検証する必要がある。

## 検証したい仮説

- Unit / Component / E2Eのテストコード生成にAIを活用することで、どこまで自動化できるか、AIのみで実務レベルの品質を担保できるかを検証できる。
- AIと人間の役割を明確にできる。
- 要件や仕様を事前にまとめた場合と、まとめていない場合で、生成されるテストケースの網羅性に差が出る。

## 対象案件

- 大和冷機（ドキュメントが整備されている）
- コーポレート（ドキュメントがまだ少ない）

## アクション

- [ ] パイロット案件の1機能について、テスト対象・期待結果・前提条件を整理する。
- [ ] 整理した情報をもとに、AIへテストコード生成を依頼する。
- [ ] AIが生成したテストコードを実行できる状態に修正する。
- [ ] AI生成コードの不足観点、誤り、過剰なテスト、保守しづらい書き方を記録する。
- [ ] 人間だけで作る場合と比べて、初期作成時間・修正量・レビュー負荷を記録する。
- [ ] AIにテストコード生成を任せてよい範囲と、人間が必ず判断すべき範囲を整理する。

## 仮説の有効さを判断する

- [ ] AIを使うことで、テストコードの初期作成時間を短縮できた。
- [ ] AI生成コードをレビュー・修正すれば、実務で使える水準にできた。
- [ ] AI生成後の修正量が許容範囲に収まる。
- [ ] 生成コードの不足観点や誤りをレビュー観点として整理できた。
- [ ] AIに任せる範囲と、人間が判断する範囲をガイドライン化できる。

## 成果物

- [ ] AI生成テストコードのサンプル
- [ ] AI生成コードのレビュー観点
- [ ] AIに渡す入力情報テンプレート
- [ ] AIによるテストコード生成の有効性検証メモ
