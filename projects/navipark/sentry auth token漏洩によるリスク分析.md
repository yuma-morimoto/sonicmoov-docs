

- エラー文言によりユーザー情報が漏れる可能性があった
- 全てのユーザーの情報が漏洩することはなかった

## sentry auth token漏洩によるリスクについて

| 漏洩情報                              | 悪用されうること                                | リスク |
| --------------------------------- | --------------------------------------- | --- |
| IPアドレス                            | おおまかな地域・回線・端末利用環境の推測、ユーザー追跡の補助          | 中   |
| `userId`                          | 社内DBや他ログと突合された場合に個人特定される                | 中〜高 |
| GPS緯度経度                           | 利用者の現在地・行動圏・生活圏の推測                      | 高   |
| Session Replay のページ遷移             | ユーザーがどの機能を使ったか、カード登録・月極申込・支払い画面を開いたかの把握 | 中〜高 |
| `/card/<ID>/...` のURL識別子          | カード登録・編集フローの内部ID推測、IDOR脆弱性がある場合の足がかり    | 中〜高 |
| `/monthly/.../<applicationId>`    | 月極申込・支払い・契約更新に関する内部ID推測、IDOR検証の足がかり     | 中〜高 |
| backend API パス                    | API構造の把握、攻撃対象エンドポイントの洗い出し               | 中   |
| Cognito / FCM / payment API の利用痕跡 | 認証・通知・決済フローの構成推測                        | 中   |
| source map / stacktrace           | フロント実装、ファイル構成、脆弱箇所の調査                   | 中〜高 |
| release SHA / version             | 脆弱なバージョンや修正状況の推測                        | 低〜中 |

## 「直接漏れている」とは言いにくいもの

- カード番号

- セキュリティコード

- パスワード

- Authorization header

- Cookie

- メールアドレス

- 氏名・住所・電話番号

Session Replay のテキストが全マスクされているなら、カード情報そのものが見えるリスクは低いです。

## 総評

全ユーザーの全情報が漏れるタイプではありません。

ただし、Sentry に記録された対象ユーザーについては、**IP・userId・位置情報・操作履歴・内部ID・API構造** が組み合わさるため、悪用時の主なリスクは **プライバシー侵害、行動追跡、IDOR調査、標的型フィッシング、API攻撃の下調べ** です。
